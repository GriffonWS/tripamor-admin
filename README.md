# Trip Amor Admin

Internal admin dashboard for Trip Amor. Reads from the same MongoDB as the
production backend (`tripamor_backend`) and shows users, trips, and key stats.

Built with **Next.js 15 App Router**, TypeScript, Tailwind, and Mongoose.

---

## 🚀 Local Setup

### 1. Install dependencies

```bash
cd "/Users/mohammedmouzzam/Danish/Trip Amor/tripamor-admin"
npm install
```

### 2. Configure environment

Copy the example and fill in real values:

```bash
cp .env.example .env.local
```

Edit `.env.local`:

```bash
# Same MongoDB URI as tripamor_backend/.env
MONGODB_URI=mongodb+srv://...

# Generate a random secret:
#   openssl rand -base64 32
AUTH_SECRET=<long-random-string>

# Plain-text password you'll use to log in
ADMIN_PASSWORD=<choose-a-strong-password>
```

### 3. Run dev server

```bash
npm run dev
```

Open <http://localhost:3000>. You'll be redirected to `/login` — sign in with
your `ADMIN_PASSWORD`.

## 📦 Deploy to Vercel (Free Tier)

This admin panel is designed to be deployed **separately** from the existing
Render backend. Recommended host: **Vercel**.

### 1. Push to its own GitHub repo

```bash
cd "/Users/mohammedmouzzam/Danish/Trip Amor/tripamor-admin"
git init
git add .
git commit -m "initial admin panel"
git branch -M main
gh repo create tripamor-admin --private --source=. --remote=origin --push
```

(Or push manually if you don't use the GitHub CLI.)

### 2. Import on Vercel

1. Go to <https://vercel.com/new>
2. Pick the `tripamor-admin` repo
3. Framework preset: **Next.js** (auto-detected)
4. Click **Environment Variables** and add:
   - `MONGODB_URI`
   - `AUTH_SECRET`
   - `ADMIN_PASSWORD`
5. Click **Deploy**

After ~2 minutes you'll get a URL like `tripamor-admin.vercel.app`.

### 3. Lock it down further (recommended)

- In Vercel project settings → **Deployment Protection** → enable **Password
  Protection** as a second layer.
- Or restrict access by IP under **Firewall**.

## 🔒 Security Notes

- The admin uses a single shared password (env var). Anyone with the password
  has full read access.
- All page routes are guarded by `src/middleware.ts` — unauthenticated requests
  redirect to `/login`.
- The session cookie is HTTP-only, signed with `AUTH_SECRET`, valid for 8 hours.
- **Do not commit `.env.local`** — it's gitignored.
- Create a **read-only MongoDB user** for the admin and use that connection
  string in `MONGODB_URI`. Limits blast radius if the panel is ever compromised.

## 📊 What's Tracked

### Dashboard
- Total users, Pro subscribers, % Pro
- New users this week
- Active users (used AI in last 30 days)
- Total trips, by status (planning / upcoming / ongoing / completed)
- Trips with AI-generated activities
- Top 5 destinations by trip count

### Users
- Email, name, auth methods (email/google/apple)
- Pro vs Free badge
- AI chat count
- Signup date
- Search by name or email

### Trips
- Destination, owner (name + email)
- Date range
- Status (with color)
- Activity count + days
- % of activities generated by AI
- Search by destination

## 🛠️ Extending

When you want to add a new section:

1. Create `src/app/(dashboard)/<name>/page.tsx` — automatically picks up the
   sidebar layout and auth guard.
2. Add it to `navItems` in `src/components/Sidebar.tsx`.
3. Query MongoDB directly from the server component (no need for an API route
   unless the page needs client-side interactivity).

For mutations (ban user, delete trip, etc.), create matching `route.ts` files
under `src/app/api/`.

## 🐛 Troubleshooting

**`MONGODB_URI is not set`** — copy `.env.example` to `.env.local` and fill in.

**Login keeps failing** — check `ADMIN_PASSWORD` in `.env.local`. Restart the
dev server after changing env vars.

**Mongoose cast errors** — the admin schemas are simplified mirrors. If the
backend adds new required fields, you may need to update `src/models/`.
